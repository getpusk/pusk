# История изменений

## [0.7.0] - 2026-04-03

### Added
- Навигация по push-уведомлениям между организациями — клик по пушу открывает нужный канал из любой организации
- Поддержка кириллицы и Unicode в именах пользователей
- Имя бота в боковой панели при наличии нескольких ботов
- Вибрация при входящем push-уведомлении (Android)
- Обязательная проверка линтером и тестами перед деплоем

### Fixed
- Push-уведомления содержат параметр организации для корректной маршрутизации
- Устранено расхождение ключей localStorage — единый префикс `pusk_`
- Кэш Service Worker обновлён до v75
- Миграция конфигурации golangci-lint под формат v2

---

## [0.6.1] - 2026-03-28

### Added
- Многоразовые инвайт-ссылки (7 дней, до 50 использований, отзыв)
- Индикаторы онлайн/отсутствия на аватарах
- Умная форма приглашения — автоопределение входа или регистрации
- Поиск организаций на сервере по аккаунту пользователя
- Глубокие ссылки в push-уведомлениях (каналы и чаты)
- API статистики организации + метрики Prometheus
- Защита владельца организации от удаления и понижения
- Переименование каналов администратором
- Компактный онлайн-статус на мобильных (2● 1○)
- Бесшовная миграция localStorage на префикс `pusk_`

### Fixed
- Push-подписки хранятся по устройствам (максимум 5), обновляются при открытии
- Безопасное обновление Service Worker без автоматической перезагрузки
- Изоляция организаций: упоминания, отметки о прочтении, FAB
- Форма входа с сохранённой организацией при истечении токена
- Логирование доставки пушей, группировка, requireInteraction для алертов

### Security
- Тесты изоляции между организациями
- Очистка токенов при переключении организации
- Push отключён в демо-организации

### CI/CD
- Пайплайн релиза: Alpine + RED OS + Astra Linux
- Cosign keyless (бинарник + SBOM)
- SBOM (SPDX-JSON), Trivy (HIGH/CRITICAL)
- golangci-lint: errcheck, staticcheck, nilerr

## [0.6.0] - 2026-03-25

### Added
- ES-модули — фронтенд разделён на 9 JS-файлов
- Service Worker: кэш оболочки, индикатор офлайна, уведомление об обновлении
- Бесконечная прокрутка истории сообщений
- Загрузка файлов в каналах (фото, видео, документы, миниатюры 320px)
- @упоминания с автодополнением и push-уведомлениями
- Отметки о прочтении в каналах
- Статус отсутствия + индикатор набора текста
- Фильтры алертов по статусу, точные временные метки
- Бейджи времени на активных алертах
- Ответы на сообщения с цитированием
- Закрепление сообщений (автор, кнопка открепления)
- Контекстное меню: редактирование, удаление, копирование, ответ
- Мастер настройки для новых организаций (3 шага)
- Переключатель организаций со счётчиками непрочитанных
- getUpdates long polling — миграция с Telegram polling-ботов
- Telegram-совместимые URL (`/botTOKEN/method`)
- Шаблоны Go для форматирования вебхуков
- Дедупликация вебхуков Alertmanager
- Prometheus /metrics (HTTP, WebSocket, сообщения)
- Структурированное логирование (slog)
- Файловые токены + хранение сообщений + квоты
- ACK → Alertmanager Silence API
- Кнопка /test-push в настройках
- Кликабельные URL и /команды в сообщениях
- Двойное нажатие «Назад» для выхода (мобильные PWA)
- Скелетон загрузки
- Двуязычный интерфейс (русский / английский)

### Security
- Отзыв JWT при смене пароля и удалении пользователя
- Защита от XSS, блокировка PIN после 5 попыток
- Валидация вебхуков на SSRF (DNS-резолв + IsPrivate)
- Файловые директории по организациям
- Ограничение частоты отправки и загрузки
- Content-Security-Policy
- Авторизация администратора на /admin/

### CI/CD
- Dependabot: Go-модули, GitHub Actions, npm, Docker
- GitHub Actions закреплены по SHA
- CodeQL, OpenSSF Scorecard
- Интеграционные тесты: Bot API, вебхуки, мультитенантность, аутентификация
- Проверка целостности фронтенда в CI

## [0.5.0] - 2026-03-20

### Added
- Ответы в каналах (командная координация)
- Кнопки ACK/Mute/Resolved на алертах
- Вебхук-эндпоинт: Alertmanager, Zabbix, Grafana, Uptime Kuma
- Умный парсер JSON-пэйлоадов (msg/message/text)
- sendMessage → канал по отрицательному chat_id
- Инвайт-ссылки (одноразовые, TTL 24 часа)
- Мультитенантность (SQLite на организацию)
- Регистрация организаций (системный бот + #general)
- Webhook Relay через WebSocket
- Лендинг с демо-чатом
- PWA-установка

### Security
- bcrypt, случайный JWT-секрет
- Rate limit 20/мин на IP
- IDOR-защита, SSRF-валидация
- XSS: только http/https ссылки
- JWT для файлов, валидация Origin (WebSocket)
- Graceful shutdown (SIGTERM → 5 сек)

### CI/CD
- CodeQL, govulncheck, Trivy
- Интеграционные тесты, OpenSSF Scorecard
- 34 E2E-теста Playwright
